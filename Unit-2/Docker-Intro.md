## Docker Hub
A centralized repository for Docker images.

## Docker Architecture
- Docker Client
- Docker Daemon
- Docker Registry (Docker Hub)
- Docker Image
- Docker Container

## Docker Life Cycle

### Stages
1. **Build** — Create a Docker image from a Dockerfile
2. **Ship** — Push the Docker image to a registry (e.g., Docker Hub)
3. **Run** — Pull the Docker image from the registry and run it as a container

### Container States
- **Created** — Container created but not started
- **Running** — Container is currently executing
- **Paused** — Container paused; no processes executing
- **Stopped** — Container stopped and not running

## Why Docker is Faster Than Virtual Machines
- **Shared kernel** — Containers share the host OS kernel vs. VMs running a full guest OS, making containers more lightweight and faster to start
- **No boot overhead** — Eliminates guest OS boot time, reducing container startup
- **Resource efficient** — Reduced resource usage and faster startup due to shared OS and efficient system resource utilization

## Docker Commands
- `docker pull ubuntu` — Pulls the Ubuntu image from Docker Hub
- `docker images` — Lists all Docker images on the local machine
- `docker run ubuntu` — Runs a container from the Ubuntu image
- `docker run -it ubuntu` — Runs a container in interactive mode
- `docker ps` — Lists all running containers
- `docker stop <container_id>` — Stops a running container
- `docker rm <container_id>` — Removes a stopped container
- `docker rmi <image_id>` — Removes a Docker image from the local machine
- `docker prune` — Removes all stopped containers, unused networks, and dangling images to free up space

## Running Nginx in a Docker Container
1. Pull the Nginx image:
```bash
docker pull nginx
```
2. Run the Nginx container:
```bash
docker run -d -p 8080:80 nginx
```
3. See running containers:
```bash
docker ps
```
4. Stop the Nginx container:
```bash
docker stop <container_id>
```
5. Remove the Nginx container:
```bash
docker rm <container_id>
```
6. Remove the Nginx image:
```bash
docker rmi nginx
```

## Container Interaction Commands
- `docker exec -it <container_id> bash` — Access the container's shell
- `docker logs <container_id>` — View logs of a container
- `docker inspect <container_id>` — Get detailed information about a container
- `docker stats <container_id>` — View real-time resource usage of a container
- `docker top <container_id>` — Display running processes in a container

## Docker Volumes
- Used to persist data generated by and used by Docker containers.
- Volumes are stored outside the container's filesystem, allowing data to persist even if the container is removed.
- Volumes can be shared between multiple containers, enabling data sharing and collaboration.

## Volume Commands
- `docker volume create <volume_name>` — Create a new Docker volume
- `docker volume ls` — List all Docker volumes
- `docker volume inspect <volume_name>` — Get detailed information about a Docker volume
- `docker volume rm <volume_name>` — Remove a Docker volume
